# Supabase Setup

Run these files manually in the Supabase SQL Editor before continuing `SESSION_005`.

## Order

1. Open Supabase project `evochia-finance`
2. Go to SQL Editor → New query
3. Paste `supabase/migrations/001_schema.sql`
4. Run
5. New query
6. Paste `supabase/migrations/002_rls_policies.sql`
7. Run

## Verify

In Table Editor you should see 6 application tables:

- `books`
- `accounts`
- `categories`
- `tags`
- `recurring_templates`
- `transactions`

For each table, RLS must be enabled.

## Notes

- `sync_outbox` and `sync_metadata` stay local-only in SQLite.
- All remote data tables have `user_id UUID REFERENCES auth.users(id)`.
- Data entity IDs are `TEXT`, not `UUID`, to match existing local IDs such as `book-business` and `cat-biz-exp-materials`.
- VAT/net fields on `transactions` are generated by Postgres.
- Do not paste or share the `service_role` key.

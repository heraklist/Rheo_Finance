## Verify

In Table Editor you should see 11 application tables:

- `books`
- `accounts`
- `categories`
- `tags`
- `recurring_templates`
- `transactions`
- `plan`
- `plan_expense_item`
- `plan_income_item`
- `coverage_expense`
- `coverage_income`

For each table, RLS must be enabled.
For each table, the `authenticated` role must have Data API table privileges.
The `anon` role should not have direct table privileges.

Storage must also have a private bucket named `receipts`. Receipt files are stored as:

```text
{user_id}/{transaction_id}.jpg
```

## Notes

- `sync_outbox` and `sync_metadata` stay local-only in SQLite.
- All remote data tables have `user_id UUID REFERENCES auth.users(id)`.
- Data entity IDs are `TEXT`, not `UUID`, because local SQLite owns IDs and newer installs rotate seed rows to install-specific UUID strings before first sync.
- VAT/net fields on `transactions` are generated by Postgres.
- Sync deletes are soft-deleted remotely via `deleted_at` tombstones so other devices can pull deletions.
- Do not paste or share the `service_role` key.

# Rheo Finance 0.2.27 release candidate checklist

## Release scope

0.2.27 is the first runtime validation build after the hardening sprint.

Included hardening areas:

- Auth session persistence and local save hardening.
- Partial sync progress without permanent red error state.
- SQLite foreign-key remap and tag `NOCASE` migration.
- Checkout contract fix.
- Receipt sync failure preservation and retry visibility.
- Settings sync diagnostics for pending, retryable, and stuck outbox rows.
- Transaction save watchdog feedback.
- Supabase local database type contract.

## Build artifacts to validate

- Android APK.
- Desktop installer.
- Desktop updater artifact / manifest.

## Pre-release checks

Run these before publishing release artifacts:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
pnpm security:secrets
pnpm security:audit
```

For local database safety checks, run only against local Supabase:

```bash
pnpm db:check
```

Do not use production Supabase keys or production database URLs during release validation.

## Manual smoke test matrix

### 1. Auth persistence

- Log in.
- Close the app.
- Reopen the app.
- Confirm the user remains logged in.
- Confirm sync does not ask for a new session immediately after restart.

### 2. Fresh install

- Install 0.2.27 on a clean device/profile.
- Start the app.
- Confirm local database initialization completes.
- Confirm default books/accounts/categories are available.
- Confirm Settings opens without runtime errors.

### 3. Update from previous version

- Install the previous release.
- Create at least one transaction and one tag.
- Install/update to 0.2.27.
- Confirm existing data remains visible.
- Confirm tag casing and tag matching are stable.
- Confirm no fake pending-sync red state remains after restart.

### 4. Initial sync

- Use an account with remote data.
- Confirm books, accounts, categories, tags, recurring templates, plans, coverage rows, and transactions pull into the app.
- Restart the app and confirm data remains visible locally.

### 5. Transaction create flow

Create and save:

- Transaction without receipt.
- Transaction with tag.
- Transaction with receipt/photo.

Confirm:

- Save button is disabled while saving.
- Dashboard updates after save.
- Transaction remains after restart.
- If save is slow, user sees clear feedback instead of a permanently locked submit state.

### 6. Sync after local changes

- Create or edit a transaction.
- Confirm sync runs after the local change.
- Confirm sync does not continuously run every 30 seconds without a local change.
- Confirm pending count returns to zero after successful sync.

### 7. Receipt sync failure handling

With network temporarily unavailable or a simulated upload failure:

- Create a transaction with a receipt/photo.
- Confirm the local transaction is preserved.
- Confirm the sync outbox keeps the failed item for retry.
- Confirm Settings shows sync diagnostics when pending/stuck items exist.

### 8. Settings sync diagnostics

Create or simulate retryable/stuck outbox rows.

Confirm Settings shows:

- Pending count.
- Retryable count.
- Stuck count.
- Entity type.
- Operation.
- Attempt count.
- Last error where available.
- Created timestamp.

### 9. Auto-update

- Install previous desktop release.
- Publish 0.2.27 updater artifact/manifest.
- Trigger update check.
- Confirm update is detected.
- Confirm download/install completes.
- Relaunch and confirm app version reads 0.2.27.

## Release decision

Release 0.2.27 only if:

- CI is green on the release readiness PR.
- Security Audit is green.
- Secrets Scan is green.
- Android APK installs and opens.
- Desktop installer installs and opens.
- Update from previous release preserves local data.
- Fresh install initializes correctly.
- Transaction save and sync smoke tests pass.
